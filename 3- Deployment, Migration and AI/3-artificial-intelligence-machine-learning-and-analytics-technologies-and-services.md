# RedShift and RedShift Serverless

## RedShift
AWS RedShift is DataWare housing service used for reporting and analytics, for BI usage.

Each of AWS services can push to data RedShift 

+ Massively Parallel Processing (MPP)
+ Automated data management, replication and scaling without downtime 
+ Designed for OLAP (great for analytics)

## RedShift Serverless

Removes the complexity of managing servers and allows user to focus on analyzing data.

    
    Use Cases:

    - complex quering and reporting
    - data lake integration
    - operation analytics (for exemple realtime decision like drivers routing for delivery business

# Kinesis

A family of services to collect, process and analyze streaming data in real time.


    Use Cases:

    - stream of financial transactions
    - stock prices
    - ioT sensors

## Kinesis Stream

Kinesis uses Shards (storage support with 24 hour rotation by default) to store stream from Producers (IoT, Social media, ..)
until the streams are processed by a Consumer (Lamba, EC2, ...) which can store the results later on a permanent storage (S3, Dynamo, RedShift, ..)

2 options are available:

- Kinesis data streams
- Kinesis video streams

## Kinesis Data Firehose

Used for capture, transform and load data streams into AWS data stores to enable near realtime analytics with BI tools.

- dynamically scales
- realtime processing
- transform, customize and enhance data before loading
- integrated monitoring with CloudWatch and error retries

Unlike Kinesis Stream, Kinesis Firehose:
        - no shards = no data retention
        - no consumers

    
    Use Cases:

    - realtime analytics
    - data lake feeding
    - IoT data integration

## Kinesis Athena

Athena is an interactive query service for data in S3

- allows to run SQL queries on data stored on S3
- serverless (no need for infra configuration)
- pay for queries

# Amazon Glue

Is a serverless data preparation and integration service

- data can be input from S3, DynamoDB, RDS, Kinesis
- discover and catalogs your data (use a crawler on data sources and creates metadata/data catalogs)
- perform ETL (Extract, Transform and Load) operations


    Used to prepare data for analytics and Machine Learning

# Amazon Data Exchange

Allows to securely use / exchange data provided from third party on subscription basis

- Data Exchange is a central way to exchange data
- data is provided in form of Data Product available from a catalogue (CSV, Parquet, image, ...)
- there are more than 3500 of Data Products available (including financial data, manufacturing, telecom, ..)
- the subscriptions last between 1 and 36 months (sometime providers give it for free)
- Data providers (Mastercard, Experian, ..) can make available Data Product through marketplace
- Own Public Data Product can be created and published, and users can Subscribe to your Data.

# Amazon Elastic Map Reduce EMR

A big data platform allowing large scale parallel data processing (PB interactive analysis)

- supports structured data (financial) / semi-structured data (text, document) / unstructured data (application logs)
- can extract data from various sources: S3, DynamoDB, RedShift, Kinesis
- supports open-source frameworks: Apache Spark, Apache Hive, Hadoop, Presto
- No need to worry about provisioning infrastructure, configuring open-source frameworks, scale capacity and performance optimization
        

    Use Cases:
    
    - Genomic data analysis: use statistical algorithms and predictive models to discover patterns and find correlation
    - Click-stream: understand customer preference and market trends
    - Logs: Process logs generated by own application

# Amazon OpenSearch


     Elasticsearch

        - is open-source data analysis tool
        - enables real time analysis to make better decisions
        - used to search security, infrastructure and applicative logs for better understanding and investigation

Amazon OpenSearch is fully-managed Elasticsearch service 
- simplify deployment and administration
- hardware provisioning and configuring Elasticsearch
- software installation and patching
- failure recovery, automated backups and monitoring
- integrates other tool ELK (Logstash, Kibana)
- integrates with AWS service (CloudWatch, CloudWatch logs, Kinesis FireHose, Lambda, S3, ...)

# Amazon Managed Streaming for Apaache Kafka MSK


     Apache Kafka

        - is widely used open-source technology to build realtime data streaming pipelines
        - process stream of events from hundreds of event sources
        - data consumers read the data and process it in the order it was produced

Amazon MSK is fully-managed Apache Kafka cluster
- hardware provisioning and configuring
- software installation and patching
- replacing unhealthy nodes, scaling up and down when needed and monitoring

# Amazon QuickSight

SaaS model business analytics service allowing you to gain value from your data

- connect with your on-premise and AWS data sources
- allows you to create dashboards, charts and diagrams
- take business data-driven decisions 

    
    Use Cases:

    - analyse sales performance (best product, rank regions, who is buying what)

# Amazon SageMaker

A fully managed Machine Learning platform 


- allowing data preparation: evaluate data quality, collect the data 
- ML model implementation: offers built-in models, supports custom ones
- train, test, tune and evaluate the ML model: optimized infrastructure for that purpose
- deploy and put to work: provide optimized EC2 instances, serverless or deploy on edge devices on premises

    
    Use Cases:

    - recommandation engine: to recommand product based on customer history
    - fraud detection
    - predicts insurance claims: detect high risk customers
    - virtual customer service assistant


# Amazon Kendra

An intelligent Search Service 

- it uses natural query language processing
- customized search
- can be plugged to different data sources (S3, FSx, RDS, Aurora, Websites, ...)
- supports unstructured and semi-structured data supports (CSV, HTML, XML, PDF, Microsoft Office)


    How it works ? 

    1- you provide and plug the data sources
    2- Kendra indexes the provided data
    3- you can ask FACT-BASED question (where, when, who) or DESCRIPTIVE (how)

# Textract, Transcribe and Translate

Group of services that uses ML to process and generate text

## Textract

Extract information from document

- read and process any type of document (images, PDF, )
- uses Optical Character Recognition (OCR = processes handwritten or printed text into machine-encoded text)


    Use Case: Automated ID recognition tool

## Transcribe

Speech to text service 
- deals with streamed audio or files (mp3)
- generates subtitles or meeting notes
    
    
    Use Cases:
    
    - Automated ID recognition tool
    - Generated meeting notes        

## Translate

Language translation service 
- accurate and natural sounding
- large scale via a single call
- supports 70+ languages, customizable with your own terms


    Use Case:

    - translate product documentation 
    - translate support documentation
    - realtime translation for customer support chatBot
